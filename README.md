# Diaspora

Мультидоменная социально-информационная платформа на **Laravel 9**, **PHP 8.1** и **MySQL**.

Один экземпляр приложения обслуживает несколько национальных диаспор. По доменному имени выбираются название сообщества, родной язык, русский язык, оформление и отдельный контент.

## Реализовано в первом MVP

- определение диаспоры по доменному имени;
- русский и узбекский интерфейс;
- регистрация пользователей 18+ по телефону, email необязателен;
- анкеты для общения, дружбы, семьи, работы и деловых контактов;
- поиск людей по городу, полу и цели знакомства;
- общая лента публикаций;
- личные диалоги и сообщения;
- каталог вакансий и отправка вакансий на модерацию;
- отметки официального оформления и предоставления жилья;
- генератор двуязычных писем по шаблонам;
- печать готового обращения или сохранение в PDF через браузер;
- раздел «Безопасность и правовая помощь»;
- сообщения о мошенничестве, насилии, удержании документов, принудительном труде, задержании и невыплате зарплаты;
- создание суперadministratora через переменные окружения;
- CI-проверка на PHP 8.1 и MySQL 8.0.

## Структура мультидиаспорности

- `diasporas` — настройки конкретной диаспоры;
- `diaspora_domains` — домены, привязанные к диаспоре;
- `supported_locales` — русский и родные языки;
- `theme` — цвета и оформление;
- все публикации, вакансии, письма и справочные материалы связаны с `diaspora_id`.

Пример:

- `uz.example.ru` → узбекская диаспора, русский + узбекский;
- `tj.example.ru` → таджикская диаспора, русский + таджикский;
- `kg.example.ru` → кыргызская диаспора, русский + кыргызский.

## Требования

- PHP 8.0.2–8.1, рекомендуемая версия — PHP 8.1;
- Composer 2;
- MySQL 5.7+ или MySQL 8.0;
- расширения PHP: `mbstring`, `openssl`, `pdo_mysql`, `tokenizer`, `xml`, `ctype`, `json`, `bcmath`, `fileinfo`.

## Установка

```bash
git clone https://github.com/asbcorp24/diaspora.git
cd diaspora
git checkout feature/laravel9-foundation

composer install
cp .env.example .env
php artisan key:generate
```

Создайте базу данных:

```sql
CREATE DATABASE diaspora CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Настройте подключение в `.env`:

```dotenv
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=diaspora
DB_USERNAME=root
DB_PASSWORD=
```

Перед первым запуском обязательно замените данные администратора:

```dotenv
ADMIN_EMAIL=admin@example.com
ADMIN_PHONE=+70000000000
ADMIN_PASSWORD=ChangeMe123!
```

После этого выполните:

```bash
php artisan migrate --seed
php artisan serve
```

Откройте `http://127.0.0.1:8000`. В режиме `local` приложение использует первую активную диаспору, даже если домен ещё не добавлен.

## Локальный домен

Добавьте в файл `hosts`:

```text
127.0.0.1 diaspora.test
127.0.0.1 uz.diaspora.test
```

Запустите сервер на этом домене через Nginx/Apache или локальную среду разработки. Сидер уже создаёт домены `diaspora.test`, `uz.diaspora.test`, `localhost` и `127.0.0.1`.

## Добавление новой диаспоры

Нужно добавить запись в `diasporas`, затем привязать к ней один или несколько доменов в `diaspora_domains`. Пользователи и весь контент автоматически изолируются по `diaspora_id`.

Для новой диаспоры потребуются:

1. код, название и название на родном языке;
2. основной и дополнительные домены;
3. список языков;
4. цветовая тема;
5. переведённые шаблоны писем;
6. проверенные материалы раздела безопасности;
7. собственные модераторы.

## Важные правила безопасности

«Криминальный раздел» реализуется только как раздел законной безопасности и защиты прав. На платформе нельзя публиковать персональные данные предполагаемых нарушителей, призывать к самосуду, продавать документы, искать незаконные услуги или размещать непроверенные обвинения. Жалобы видят модераторы; публичная публикация допускается только после проверки и обезличивания.
